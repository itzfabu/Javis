# Jarvis-as-a-Service Pricing Research

## Summary

**Thumbs down** on selling Jarvis as a productized, marketplace-listed setup service at premium pricing. **Thumbs up** on positioning it as a demo-driven lead magnet for bespoke consulting sourced outside Fiverr/Upwork.

"Claude Code setup" is already a commoditized search term on Fiverr, priced at **$15–$45** by several near-identical, keyword-stuffed gig listings. Jarvis's real differentiators (persistent knowledge graph, multi-agent delegation, custom orb dashboard) aren't visible or provable to a stranger browsing a marketplace, and the current build is personal-scale (no auth, no multi-tenant handling) — not yet robust enough to defend a high price against buyer scrutiny.

## Key details

**Pricing found by category (Fiverr/Upwork, July 2026):**
- "Claude Code setup" / "Claude AI agent" gigs (closest match): **$15–$45**, one outlier at $175 — reads as a flooded, race-to-the-bottom sub-niche
- AI automation (n8n/Zapier/Make + AI agents), framed around integrations/ROI: **$80–$500**
- Custom GPT / "AI assistant ecosystem" builds (Fiverr's own cost guide): $50–$200 (basic) up to **$500–$2,000** for a full multi-flow ecosystem — closest realistic ceiling for what Jarvis actually is
- Upwork: hourly $30–$150 (median ~$45/hr for chatbot specialists); fixed-price custom builds **$1,000–$5,000**; ongoing retainers **$2,000–$8,000/month**

**Why the price gap exists:** gigs priced around vague "AI assistant setup" anchor low; gigs framed around concrete automations/integrations with measurable outcomes anchor 3–10x higher. Framing, not just capability, drives price.

**Differentiation — real vs. not yet provable:**
- Real: persistent structured knowledge system (PROJECTS/GOALS/TASKS + graph sync), multi-agent delegation, a working visual dashboard — none of the sub-$100 gigs have this.
- Not yet provable: none of it is visible to a cold marketplace buyer without a demo. "Knowledge graph" and "multi-agent" read as buzzwords unless demonstrated live.
- Not yet ready: the underlying implementation is single-user/personal-scale — productizing it for third-party clients means real hardening work (auth, config isolation, error handling) that isn't priced into any "package it and sell it" plan yet.

**Data confidence caveat:** Fiverr blocked direct page fetches (403 on every gig URL), so all figures come from WebSearch snippet aggregation, not verified live listings with confirmed review counts/seller tiers. Treat prices as directionally reliable, not precise — a manual logged-in browse of 15–20 live gigs is the recommended way to firm this up.

**Recommended next steps (from the research):**
1. Manually verify pricing via a logged-in browse of live competitor gigs (WebFetch was blocked).
2. Build a 3–5 min demo video of the orb + agent delegation — highest-leverage step, since differentiation is invisible without seeing it work.
3. Pick target buyer: individual assistant-seekers (low margin, per data) vs. small-business automation buyers (better margin, more receptive to demo-driven pitches) — data favors the latter.
4. Price-test outside Fiverr/Upwork first (a few free/discounted pilot builds for a testimonial) before listing publicly, to avoid anchoring against the $15–$45 crowd.
